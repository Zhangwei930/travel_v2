1. 项目定位与重建目标
项目名称建议：周密出游智能攻略系统。
对外定位：帮助用户快速解决“今天去哪玩、周末怎么玩、附近怎么安排路线”的问题。
后台定位：小程序 + 地图 API + Dify 工作流 + 本地 AI + WebSearch + Obsidian 知识库 + 数据库/向量库。
1.1 不再做普通旅游攻略大全
普通旅游攻略赛道已经被高德、美团、小红书、携程、大众点评等平台覆盖。新版项目不能正面竞争“全网攻
略”和“交易预订”，而应切入“附近、即时、场景化、路线组合”的需求。
平台/能力 主要解决的问题 周密出游应避开的正面竞争
高德/腾讯地图 在哪里、怎么去、附近有什么 不要自建地图，不要做导航替代
美团/大众点评 吃什么、怎么买、团购优惠 不要主打交易和团购
小红书/抖音 别人怎么玩、内容种草 不要依赖抓取内容做攻略
携程/飞猪 酒店、机票、门票预订 不要一开始做大旅游交易
周密出游 我现在适合怎么玩、怎么安排半天/一天 主打场景化路线与本地知识闭环
1.2 核心用户问题
 今天附近去哪玩？
 周末带孩子去哪不累？
 下雨天适合去哪？
 预算低怎么安排一日游？
 情侣约会怎么安排路线？
 几个地点怎么串成一条顺路的路线？
 这个地方停车、排队、门票、营业状态需要注意什么？
1.3 产品核心卖点
 一键生成可执行出游方案，而不是只列一堆景点。
 结合地图 API 的真实位置、距离和路线能力。
 结合 Obsidian/数据库沉淀的本地玩法、场景标签、避坑经验。
 知识库不足时自动触发 WebSearch，生成待审核内容。
 用户反馈会回流，形成自己的数据资产。
2. 总体架构设计
系统采用“前台小程序 + 业务后端 + Dify 工作流 + 地图 API + 本地/云端知识库 + 本地 AI”的分层架构。

4. 地图 API 接入与入库边界
地图 API 是系统必须接入的一层。它解决真实地点、距离、坐标、路线、导航等问题，但地图数据不应被当成
自己的长期原始地图数据库。
4.1 地图 API 负责什么
 POI 搜索：根据关键词查地点。
 附近搜索：根据经纬度查附近景点、商场、公园、餐饮等。
 地理编码/逆地理编码：地址与经纬度互转。
 距离计算：判断地点之间是否顺路。
 路线规划：步行、驾车、公交等路线建议。
 实时数据：交通、路线耗时、天气等需要实时调用。
4.2 可以写入本地库的数据
字段 是否写入 建议
POI 名称 可以 作为地点索引，不代表拥有地图数据
平台 POI ID 可以 用于后续刷新和重新查询
经纬度 谨慎保存 用于本地匹配和距离计算，标记来源与更
新时间
地址 谨慎保存 标记来源与更新时间
分类 可以 用于初步筛选，如博物馆、公园、商场
路线耗时/实时交通 不建议长期保存 每次实时调用或短期缓存
完整地图返回 JSON 不建议 容易产生合规和过期问题
你的推荐理由/标签/避坑 必须保存 这是自己的核心知识资产

8. WebSearch 联网补充机制
WebSearch 用于补足“知识库没有”或“需要最新信息”的问题。推荐先使用 SearXNG 自建搜索，避免一开
始使用付费搜索 API。
8.1 触发条件
 知识库相似度低于阈值。
 用户询问最新活动、新开业、临时闭馆、节假日安排。
 地点缺少适合人群、开放状态、注意事项等信息。
 用户反馈指出现有攻略过期或错误。
8.2 搜索处理流程
搜索 Query 生成
↓
SearXNG / WebSearch 获取结果
↓
提取标题、摘要、链接、发布时间或更新时间
↓
去重、过滤低质量来源
↓
AI 总结为候选答案
↓
风险等级判断
↓
写入 search_cache 和 kb_pending
↓
生成 Obsidian 待审核 Markdown
8.3 风险分级
内容类型 处理方式
普通景点介绍、玩法建议 可自动生成草稿，人工抽查
营业时间、票价、停车、临时活动 必须标记来源和更新时间，建议人工审核
安全风险、政策限制、退款、支付、平台规则 必须人工确认后入库
用户投诉、商家合作、价格报价 不自动回复，转人工处理

14. 成本控制、风险与合规边界
14.1 0 成本优先组件
组件 成本策略
Obsidian 本体免费，本地硬盘保存资料
Ollama 本地运行免费，主要消耗本机算力和电费
SearXNG 自建免费搜索，避免一开始用付费搜索 API
Dify 自部署版本可控，适合先做工作流
PostgreSQL / Qdrant / Redis 开源自部署
Oracle Cloud 尽量使用免费资源，但注意配额和流量限制
地图 API 关注免费额度，超额可能产生费用
14.2 需要控制的风险
 地图数据合规：不要把地图 API 返回数据批量抓取并永久当作自己的地图库。
 AI 幻觉：营业时间、票价、路线耗时不能由 AI 编造。
 知识污染：联网搜索结果必须进入待审核流程。
 成本失控：搜索 API、地图 API、云服务和大模型 API 都可能超额收费。
 内容过期：营业时间、临时活动、闭馆信息需要设置更新时间和过期策略。
 用户安全：夜游、户外、偏远地点推荐需要加入安全提醒。
